SCSS 解説ガイド

対象テーマ: polartech-2024　／　assets/scss/global/ の内容

このガイドについて

SCSSをこれから触る人向けに、このプロジェクトで使っている共通設定ファイルの読み方・使い方を解説します。

SCSSとは？

SCSSはCSSをより書きやすくした「拡張言語」です。変数・繰り返しの省略・条件分岐など、CSSにはない便利な機能が使えます。

ブラウザはSCSSを直接読めないため、Gulpでコンパイル（変換）してCSSにしてから使います。

global フォルダの役割

assets/scss/global/ はサイト全体で使い回す「共通設定」をまとめた場所です。直接スタイルを書くのではなく、他のSCSSファイルから呼び出して使います。

ファイル内容
_variables.scss色・フォント・サイズなどの定数（変数）をまとめた設定ファイル
_mixin.scssよく使うCSSのまとまりに名前をつけて再利用できるようにしたもの
_functions.scss計算処理に名前をつけて再利用できるようにしたもの
_index.scss上記3ファイルをまとめて外部に公開するためのファイル（触らなくてOK）
ファイル名が _ から始まる理由
アンダースコアで始まるSCSSファイルは「パーシャル」と呼ばれ、単体でCSSに変換されません。他のファイルから @use や @forward で読み込まれる専用のファイルです。
_variables.scss

変数（Variables）

色・フォント・余白などの値に「名前」をつけて管理するファイルです。同じ値を何度も書かずに済み、変更するときは1箇所直すだけで全体に反映されます。

なぜ変数を使うのか

変数なし（修正が大変）
color: #142633;
background: #142633;
border-color: #142633;
変数あり（1箇所直すだけ）
color: $c-main;
background: $c-main;
border-color: $c-main;

カラーコードを後から変更したい場合、変数を使っていれば $c-main: #142633; の1行を書き換えるだけで全体が更新されます。

カラー変数

$c-sub1 #E5F6FE
$c-sub2 #F3F8FA
$c-sub3 #E8CF28
$c-main #142633
$c-accent #25445A
_variables.scss
// Color
$c-sub1: #E5F6FE;   // 薄い水色（背景などに使う補助色）
$c-sub2: #F3F8FA;   // さらに薄い青白（背景などに使う補助色）
$c-sub3: #E8CF28;   // 黄色（アクセント補助色）
$c-main: #142633;   // ネイビー（メインカラー）
$c-accent: #25445A; // ダークブルー（アクセントカラー）

フォント変数

_variables.scss
// Font
$f1: "Noto Sans JP", sans-serif;  // 日本語メインフォント
$f2: "Montserrat", sans-serif;     // 英字フォント（見出しなどに使う）
$fontawesome: "Font Awesome 6 Free"; // アイコンフォント
使い方（例）
font-family: $f1;
font-family: $f2;

サイズ・余白変数

_variables.scss
// ヘッダーの高さ
$pcHeaderHeight: 100px;
$spHeaderHeight: 70px;

// コンテンツ幅の上限（S/M/Lの3段階）
$pcMaxWidthS: 960px;
$pcMaxWidthM: 1180px;
$pcMaxWidthL: 1520px;

// コンテンツ左右のpadding（PC）
$pcInnerPadding: 40px;

// セクション上下のpadding/margin
$pcSectionPadding: 96px;  // PC・上下余白
$pcSectionMargin: 80px;   // PC・要素間の距離
$spSectionPadding: 48px;  // SP・上下余白
$spSectionMargin: 40px;   // SP・要素間の距離

// ドロップシャドウ（影）
$drop-shadow: 3px 3px 10px rgb(0 0 0 / 0.05);
使い方（例）
max-width: $pcMaxWidthM;
padding: $pcSectionPadding $pcInnerPadding;
box-shadow: $drop-shadow;
_mixin.scss

Mixin（ミックスイン）

よく使うCSSのまとまりに名前をつけたものです。@include 名前 と書くだけで、あらかじめ定義したCSSをまるごと呼び出せます。

Mixinを使う前の準備
Mixinを使いたいSCSSファイルの先頭に @use "global" as var; と書く必要があります。その後 var.mixin名() の形で呼び出せます。

mq()　―　メディアクエリ（レスポンシブ対応）

画面サイズによって表示を変えるCSSを書くための mixin です。ブレークポイント（切り替え幅）に名前がついているので、数値を覚えなくても使えます。

引数対象画面幅用途
sp520px 以下スマートフォン向けスタイル
tab960px 以下タブレット・スマートフォン向けスタイル
nav1366px 以下ナビゲーションの切り替え（ハンバーガーメニューなど）
Mixinなし（毎回長い記述が必要）
@media screen and (max-width: 520px) {
  .title {
    font-size: 18px;
  }
}
Mixinあり（すっきり書ける）
@include var.mq(sp) {
  .title {
    font-size: 18px;
  }
}

bg_image()　―　背景画像の設定

背景画像を設定するときの定型スタイル（画像パス・繰り返しなし・中央配置）をまとめた mixin です。

使い方（例）
.hero {
  @include var.bg_image("../img/hero.jpg") {
    background-size: cover;  // {} の中に追加スタイルを書ける
  }
}
出力されるCSS
.hero {
  background-image: url("../img/hero.jpg");
  background-repeat: no-repeat;
  background-position: center;
  background-size: cover;
}

hover　―　PC・タッチデバイスのホバー出しわけ

マウスがある端末（PC）では :hover、タッチデバイス（スマホ・タブレット）では :active に自動で振り分けてくれる mixin です。

使い方（例）
.btn {
  opacity: 1;

  @include var.hover {
    opacity: 0.7; // PCではホバー時、スマホでは押したとき
  }
}
なぜ出しわけが必要？
スマートフォンは厳密には「ホバー」できません。:hover だけ書くとスマホでタップ後にスタイルが残ってしまうことがあります。この mixin を使うと両方に適切に対応できます。

line-clamp()　―　テキストを指定行数で切り詰める

長いテキストを指定した行数で切り詰めて「...」で省略表示するための mixin です。

引数動作
11行で収まらない場合は「...」で省略（横1行）
2 以上指定行数で折り返し、超えた部分を「...」で省略
1行で省略
.title {
  @include var.line-clamp(1);
}
3行で省略
.summary {
  @include var.line-clamp(3);
}
_functions.scss

Function（ファンクション）

Mixinと似ていますが、Functionは「計算して値を返す」のが目的です。プロパティ: 関数名(引数) の形で使います。

rem()　―　pxをremに変換する

デザインカンプなどで指定された px の数値を、アクセシビリティに配慮した rem 単位に自動変換してくれる関数です。

remとは？
rem はブラウザの基本フォントサイズ（通常16px）を基準にした単位です。ユーザーがブラウザのフォントサイズ設定を変えたとき、px は固定のままですが rem はそれに合わせて拡縮します。アクセシビリティ対応として推奨されます。
引数返す値
161rem（16px ÷ 16 = 1）
241.5rem（24px ÷ 16 = 1.5）
322rem（32px ÷ 16 = 2）
使い方
.title {
  font-size: var.rem(24);
  margin-bottom: var.rem(32);
}
出力されるCSS
.title {
  font-size: 1.5rem;
  margin-bottom: 2rem;
}

calclamp()　―　画面幅に応じてフォントサイズをなめらかに変化させる

スマートフォン（520px）とPC（1366px）の間で、フォントサイズや余白を滑らかに変化させるための関数です。「SP用のサイズ」と「PC用のサイズ」を渡すだけで自動計算してくれます。

clampとは？
CSSの clamp(最小値, 推奨値, 最大値) 関数を使っています。画面幅が小さいときは最小値、大きいときは最大値に固定され、その間はなめらかに変化します。
使い方（SP:16px ↔ PC:24px）
.title {
  font-size: var.calclamp(16, 24);
}
イメージ（画面幅による変化）
// 520px以下  → 16px固定
// 520〜1366px → 徐々に拡大
// 1366px以上 → 24px固定
ポイント
引数は px数値のみ（単位なし） で渡してください。rem() と組み合わせて内部で自動的に rem に変換されます。

実際のSCSSファイルでの使い方

global フォルダの変数・mixin・function を自分のSCSSファイルで使うには、まず先頭に @use を書く必要があります。

使いたいSCSSファイルの先頭に書く
@use "global" as var;
as var は「globalを var という名前で呼び出す」という意味です。var.変数名 / var.mixin名() / var.関数名() の形で使います。
まとめて使う例
@use "global" as var;

.card {
  font-family: var.$f1;
  color: var.$c-main;
  font-size: var.rem(16);
  box-shadow: var.$drop-shadow;

  .card__title {
    font-size: var.calclamp(18, 24);
    font-family: var.$f2;
    @include var.line-clamp(2);
  }

  @include var.hover {
    opacity: 0.8;
  }

  @include var.mq(sp) {
    padding: var.rem(16);
  }
}
