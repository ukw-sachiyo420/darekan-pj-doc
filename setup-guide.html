Windows向け 開発環境構築ガイド

対象テーマ: polartech-2024　／　作成日: 2026-06-30

このガイドについて

このドキュメントは、Windows PCで初めてこのテーマの開発環境を整える人向けの手引きです。

Node.js・npm・Gulpを知らなくても、STEP 0から順番に進めれば動かせるように書いています。

そもそも何をするのか（全体像）

このテーマではスタイルを「SCSS」という形式で書いています。ブラウザはSCSSを直接読めないため、CSSに変換（コンパイル）する作業が必要です。この変換をファイル保存のたびに自動でやってくれるのが、今回セットアップするツール群です。

ツール 役割
Node.js JavaScriptをPC上で動かすための実行環境。GulpなどのツールはNode.js上で動く。
npm Node.jsと一緒に入るパッケージ管理ツール。必要なツールを一括でインストールできる。
Gulp タスクランナー。「SCSSを監視してCSSに変換する」などの自動化処理を実行する。
BrowserSync ファイルを保存したときにブラウザを自動リロードしてくれるツール。

作業場所：VSCodeのターミナル（PowerShell）

このガイドのコマンドは、VSCodeの統合ターミナルで実行します。外部アプリは不要でVSCode内で完結します。

ターミナルの開き方

  1. VSCodeのメニューから「ターミナル → 新しいターミナル」を選ぶ
  2. ターミナル右上のシェル選択（+ の横にある ∨ ボタン）から PowerShell を選ぶ
  3. 以降の手順はすべてこのターミナルで実行する
PowerShellが選択肢にない場合は、Windowsの検索から「PowerShell」を開いて作業しても問題ありません。
STEP 0

Node.jsが入っているか確認する

まずNode.jsがすでに導入済みかどうかを確認します。ターミナルで次の2行を1行ずつ実行してください。

node -v
npm -v
✅
バージョン番号が表示される（例: v20.11.0） Node.js導入済みです。STEP 2へ進んでください。 （Node.jsが既に入っている場合、STEP 1はスキップしてSTEP 2へ）
❌
「node は認識されません」のようなエラーが出る 未導入です。STEP 1へ進んでください。
STEP 1

Node.jsをインストールする

インストール手順

  1. ブラウザで Node.js 公式サイト（https://nodejs.org/ja）を開く
  2. 「LTS（推奨版）」と書かれたボタンからインストーラー（.msi ファイル）をダウンロードする
  3. ダウンロードしたファイルをダブルクリックして起動する
  4. インストーラーはすべてデフォルト設定のまま「Next」→「Install」で進めてよい
  5. 完了したら VSCodeを完全に閉じ、再度起動する（ターミナルを閉じるだけでは不十分）

インストール後の確認

VSCodeを再起動したあと、ターミナルで再度実行:

node -v
npm -v
両方ともバージョン番号が表示されればOKです。まだエラーになる場合はPCを再起動してから再確認してください。

うまく入らない場合

  • 会社PCで「管理者として実行」が必要なことがある（インストーラーを右クリック→「管理者として実行」）
  • セキュリティソフトがブロックしている場合は一時的に無効化して試す
  • 古いNode.jsが入っている場合はWindowsの「アプリ」設定からアンインストール後に再インストール
STEP 2

プロジェクトファイルをテーマフォルダに配置する

担当者から共有された以下の2ファイルを、テーマフォルダ（polartech-2024）の直下に置いてください。

ファイル名 役割
package.json このプロジェクトで使うツールの一覧が書かれた設定ファイル。npm install はこのファイルを読んでインストールを行う。
gulpfile.js Gulpの動作設定ファイル。「どのSCSSを監視して、どこに出力するか」などが書かれている。

配置後のフォルダ構成（確認）

polartech-2024/
├── package.json       ← ここに置く
├── gulpfile.js        ← ここに置く
├── assets/
├── functions.php
└── ...（その他のテーマファイル）
この2ファイルがない状態で次のSTEPに進んでも npm install が正しく動きません。必ず先に配置してください。
STEP 3

必要なツールをインストールする （初回のみ）

Node.jsが入ったら、このテーマで使うツール類を一括でインストールします。npm install というコマンドがこれを自動でやってくれます。

1. VSCodeでテーマフォルダを開く

  1. VSCodeを起動する
  2. メニューから「ファイル → フォルダーを開く」を選ぶ
  3. polartech-2024 フォルダを選択して開く
  4. メニューから「ターミナル → 新しいターミナル」を開く
VSCodeでフォルダを開いた状態でターミナルを起動すると、自動的にそのフォルダの中にいる状態になります。

2. インストールを実行する

npm install
大量のテキストが流れますが正常です。エラーなく終われば完了。node_modules フォルダが生成されていれば成功です。
この手順は初回のみです。2回目以降は不要です。
STEP 4

ローカルサイトのURLを確認・設定する

このテーマではブラウザ自動リロード（BrowserSync）のためにローカルサイトのURLを指定する必要があります。

確認場所: gulpfile.js の以下の行

proxy: "http://polartech.local/",

Localアプリで自分のサイトを起動したとき表示されるURL（例: http://polartech.local/）とこの値が一致していないと、ブラウザが自動で開きません。

URLが違う場合はこの行を自分のURLに書き換えてから次のSTEPへ進んでください。
STEP 5

開発を開始する

作業するたびに毎回このSTEPを行ってください。
PCを起動して開発を始めるとき、必ず npm start を実行してからSCSSを編集してください。
実行しないままSCSSを編集しても、CSSに反映されません。

起動手順

  1. Localアプリを起動し、WordPressサイトをStartする
  2. VSCodeでテーマフォルダを開く
  3. ターミナルを開き、以下を実行する
npm start

成功するとターミナルが監視状態になり、ブラウザが自動で開きます。SCSSを編集して保存すると、CSSへの変換とブラウザリロードが自動で行われます。

停止するには: Ctrl + C　／　次回また作業するときは再度 npm start を実行してください。

SCSSの編集場所

開発中に触るのは assets/scss/ 配下のファイルです。
assets/css/ はビルド後の出力先なので直接編集しないでください。

フォルダ 内容
assets/scss/global/ 変数・mixin（サイト全体で使う共通設定）
assets/scss/base/ リセットCSSなど基盤スタイル
assets/scss/common/ ヘッダー・フッターなど共通パーツのスタイル
assets/scss/page/ 各ページ固有のスタイル
assets/scss/style.scss 全SCSSをまとめてimportするエントリファイル

トラブル対処

npm install でエラーになる

node_modules を削除してから再実行してください。

Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json
npm install
npm start でブラウザが開かない・同期しない
  • Localアプリでサイトが起動しているか確認する
  • gulpfile.js の proxy のURLが正しいか確認する
  • 上記を直したうえで npm start を再実行する
ファイルを保存してもスタイルが変わらない
  1. ターミナルにSCSSのエラーが出ていないか確認する（赤文字が出ていればSCSSの記述ミス）
  2. ブラウザでハードリロードを試す（Ctrl + F5）
  3. WordPressのキャッシュプラグイン（wp-fastest-cache）のキャッシュをクリアする
node -v は通るが npm -v でセキュリティエラーになる

PowerShellのスクリプト実行ポリシーが制限されているのが原因です。
エラーメッセージに「スクリプトの実行が無効」「.ps1 を読み込むことができません」と出ていれば確定です。

ターミナルで次を実行してください:

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

確認プロンプトが出たら Y を入力してEnterを押す。
その後ターミナルを開き直して npm -v を再確認してください。

このコマンドは「自分のユーザーアカウントの範囲でのみ、署名済みスクリプトの実行を許可する」設定です。PC全体への影響はありません。
node -v は通るが npm -v でエラーになる（その他）

VSCodeを再起動して再確認。それでも解決しない場合はPCを再起動する。

パスの途中にスペースがあって cd できない

cd のパスをダブルクォートで囲めば動きます。

cd "C:\Users\name\Local Sites\..."
